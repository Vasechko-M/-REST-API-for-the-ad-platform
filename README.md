# Ads Application

REST API для платформы объявлений: регистрация пользователей, работа с профилем, создание объявлений, загрузка изображений и управление комментариями.

## Что реализовано

- регистрация и логин пользователя;
- хранение и обновление профиля пользователя;
- CRUD для объявлений;
- загрузка и выдача изображений объявлений и аватаров;
- добавление, редактирование и удаление комментариев;
- разграничение прав доступа (автор/админ/пользователь).

## Технологии и библиотеки

- `Java 11+`
- `Spring Boot 2.7.15`
- `Spring Web` (REST-контроллеры)
- `Spring Data JPA + Hibernate` (доступ к БД)
- `Spring Security` (Basic Auth, роли, доступ к ресурсам)
- `Flyway` (миграции схемы БД)
- `PostgreSQL` (основная БД)
- `H2` (тестовый профиль)
- `MapStruct` (маппинг entity <-> DTO)
- `Lombok` (сокращение шаблонного кода)
- `springdoc-openapi` (Swagger UI/OpenAPI)
- `JUnit 5 + Spring Boot Test + spring-security-test` (тесты)

## Архитектура и как работают основные функции

Приложение построено по слоистой схеме:

- `controller` — принимает HTTP-запросы, валидирует вход и отдает DTO;
- `service` — бизнес-логика (правила доступа, сценарии работы с объявлениями, пользователями, комментариями);
- `repository` — доступ к данным через JPA;
- `entity` — ORM-модель таблиц;
- `mapper` — преобразование между сущностями и DTO;
- `config/component/filter` — безопасность, CORS, инициализация.

Основные пользовательские сценарии:

1. **Регистрация/логин**  
   Запрос в `AuthController` -> `AuthService` -> сохранение пользователя и роли -> последующая аутентификация через Spring Security.
2. **Создание объявления**  
   `AdvertisementController` принимает `multipart/form-data` (данные + файл) -> `AdService` сохраняет изображение и запись объявления с автором.
3. **Просмотр/редактирование/удаление объявления**  
   Доступ проверяется в сервисном слое и security-компонентах (`AdSecurity`), редактировать может владелец или админ.
4. **Комментарии**  
   `CommentController` вызывает `CommentService`: комментарии привязываются к объявлению и автору, изменения доступны по правам.

## Модель данных (таблицы и связи)

Миграция: `src/main/resources/db/migration/V1__init_schema.sql`.

- `roles` — справочник ролей (`id`, `name`).
- `users` — пользователи (`email`, ФИО, телефон, аватар, пароль, `role_id`).
- `advertisements` — объявления (`title`, `description`, `price`, `image`, `author_id`).
- `comments` — комментарии (`text`, `created_at`, `author_id`, `ad_id`).
- `advertisement_images` — дополнительные изображения объявления (`ad_id`, `path`).

Связи:

- `roles (1) -> (N) users`
- `users (1) -> (N) advertisements`
- `users (1) -> (N) comments`
- `advertisements (1) -> (N) comments`
- `advertisements (1) -> (N) advertisement_images`

## Миграции и запуск

1. Настройте БД в `application.properties`/`application.yml`.
2. При старте приложения Flyway автоматически применит миграции.
3. Запуск:
   - `mvn clean spring-boot:run`
4. Swagger UI обычно доступен по адресу:
   - `http://localhost:8080/swagger-ui/index.html`

## Описание проекта

Проект представляет собой RESTful API для управления пользователями, объявлениями и комментариями. Основные возможности:

- управление профилем (обновление данных, смена пароля, загрузка аватара);
- работа с объявлениями: создание, просмотр, редактирование, удаление, загрузка изображений;
- управление комментариями к объявлениям (добавление, редактирование, удаление);
- получение информации о текущем пользователе и его объявлениях.

## Быстрый старт

Чтобы быстро развернуть и запустить проект, выполните следующие шаги:

1. **Клонирование репозитория:**
   ```bash
   git clone https://github.com/Lutrix79/example-for-graduate-work.git
   cd project-name
   ```

2. **Установка зависимостей бэкенда:**
   ```bash
   ./mvnw clean install
   ```

2. **Создание базы данных PostgreSQL:**
   - Убедитесь, что PostgreSQL установлен на вашей системе.
   - Запустите терминал и подключитесь к PostgreSQL:
     ```bash
     psql -U postgres
     ```
   - Создайте новую базу данных:
     ```sql
     CREATE DATABASE example_for_graduate_work;
     ```
   - Создайте пользователя с правами на доступ к базе данных:
     ```sql
     CREATE USER adm WITH PASSWORD '123321';
     GRANT ALL PRIVILEGES ON DATABASE example_for_graduate_work TO adm;
     ```
   - Выход из терминала PostgreSQL:
     ```sql
     \q
     ```

3. **Настройка базы данных:**
   Проект использует миграции Flyway, настройте подключение в `src/main/resources/application.properties`:
   ```properties
   spring.datasource.url=jdbc:postgresql://localhost:5432/example-for-graduate-work
   spring.datasource.username=adm
   spring.datasource.password=123321
   ```

4. **Запуск бэкенда:**
   ```bash
   ./mvnw spring-boot:run
   ```

5. **Запуск фронтенда через Docker:**
   Убедитесь, что Docker установлен на вашей системе. Затем выполните команду:
   ```bash
   docker run -p 3000:3000 --rm ghcr.io/dmitry-bizin/front-react-avito:v1.21
   ```

Теперь ваше приложение доступно:
- **Бэкенд:** `http://localhost:8080`
- **Фронтенд:** `http://localhost:3000`

## Структура API

### Пользователи

| **Метод** | **Путь** | **Описание** |
|---|---|---|
| POST | /users/set_password | Смена пароля пользователя |
| GET | /users/me | Получение информации о текущем пользователе |
| PATCH | /users/me | Обновление данных пользователя |
| PATCH | /users/me/image | Загрузка или обновление аватара |

### Регистрация и авторизация

| **Метод** | **Путь** | **Описание** |
|---|---|---|
| POST | /register | Регистрация нового пользователя |
| POST | /login | Авторизация пользователя |

### Объявления

| **Метод** | **Путь** | **Описание** |
|---|---|---|
| GET | /ads | Получение всех объявлений |
| POST | /ads | Создание нового объявления (с изображением) |
| GET | /ads/{id} | Получение информации об объявлении по ID |
| PATCH | /ads/{id} | Обновление объявления |
| DELETE | /ads/{id} | Удаление объявления |
| PATCH | /ads/{id}/image | Обновление изображения объявления |
| GET | /ads/me | Получение объявлений текущего пользователя |

### Комментарии

| **Метод** | **Путь** | **Описание** |
|---|---|---|
| GET | /ads/{id}/comments | Получение комментариев к объявлению |
| POST | /ads/{id}/comments | Добавление комментария к объявлению |
| PATCH | /ads/{adId}/comments/{commentId} | Редактирование комментария |
| DELETE | /ads/{adId}/comments/{commentId} | Удаление комментария |

## Модели данных

В проекте используются следующие основные сущности:

- **User** — информация о пользователе.
- **Ad** — объявление (товар/услуга).
- **Comment** — комментарий к объявлению.

Для каждой сущности определены схемы валидации и структуры запросов/ответов согласно OpenAPI.
